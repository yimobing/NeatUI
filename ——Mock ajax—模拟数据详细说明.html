一、Mock.js 简介

Mock.js 是一款模拟数据生成器，可以帮助前端开发者在后端接口未完成时，模拟接口数据，实现前后端并行开发。

主要功能

  • 拦截 Ajax 请求：拦截前端发起的请求，返回模拟数据
  • 生成随机数据：通过数据模板生成各种类型的随机数据
  • 无需修改代码：侵入性小，只需在项目中引入即可

安装方式

// npm 安装
npm install mockjs

// 或直接引入 CDN
<script src="https://cdn.bootcdn.net/ajax/libs/Mock.js/1.0.1-beta3/mock-min.js"></script>

工作原理

Mock.js 通过重写（覆盖）原生的 XMLHttpRequest 对象，拦截所有 Ajax 请求。当请求 URL 匹配到已定义的 mock 规则时，直接返回模拟数据，不再发送真实请求。

二、基本使用方法

2.1 基本语法

Mock.mock(url, method, template|function)
参数 类型 说明
url String | RegExp 需要拦截的 URL，支持正则匹配
method String 请求方法：get、post、put、delete 等
template Object | String 数据模板，用于生成模拟数据
function Function 自定义函数，返回模拟数据

2.2 简单示例

import Mock from 'mockjs';

// 定义 mock 数据
Mock.mock('/api/user', 'get', {
    code: 200,
    message: 'success',
    data: {
        'id|1-100': 1,
        'name': '@cname',
        'age|18-60': 1,
        'email': '@email'
    }
});

// 发起请求时会被拦截
fetch('/api/user')
    .then(res => res.json())
    .then(data => console.log(data));

// 输出示例：
// {
//     code: 200,
//     message: 'success',
//     data: { id: 42, name: '张三', age: 28, email: 'zhangsan@example.com' }
// }

2.3 使用函数返回数据

Mock.mock('/api/user', 'post', (options) => {
    // options.body 为 POST 请求的数据
    let body = JSON.parse(options.body);
    
    return {
        code: 200,
        message: 'success',
        data: {
            id: body.id || 1,
            name: '测试用户'
        }
    };
});

2.4 数据模板语法

语法 说明 示例
'name|min-max': value 生成 min 到 max 个 value 'list|1-10': [{}]
'name|count': value 生成 count 个 value 'list|3': [{}]
'name|+1': value 属性值自动加 1 'id|+1': 1

2.5 常用占位符

占位符 说明 示例
@cname 中文名 张三
@name 英文名 John Smith
@email 邮箱 test@example.com
@phone 手机号 13800138000
@id 身份证号 110101199001011234
@city 城市 北京市
@date('yyyy-MM-dd') 日期 2026-04-03
@datetime 日期时间 2026-04-03 10:30:00
@image('200x100') 图片 URL 随机生成的图片地址
@paragraph 段落文本 随机生成一段中文

三、常见问题

3.1 为什么断网还能返回数据？

问题描述：使用 mock.js 模拟数据后，断开网络，接口依然能返回数据，无法测试网络断线场景。

问题原因

Mock.js 在浏览器端拦截了 XMLHttpRequest，请求根本没有发送到网络层。当 URL 匹配到 mock 规则时，直接返回模拟数据，不会发送真实的网络请求。

工作原理图解

┌─────────────────────────────────────────────────────────────────┐ │ 浏览器 │ │ │ │ 代码发起请求 ──→ Mock.js 拦截 ✋ ──→ 直接返回模拟数据 │ │ │ │ │ │ ❌ 请求被拦截，没有真正发出！ │ │ ▼ │ │ 网络层（根本没走到这里） │ │ │ └─────────────────────────────────────────────────────────────────┘
重要提示：Mock.js 拦截的是整个网络请求，所以无论是否有网络，只要匹配到 mock 规则，都会返回模拟数据。这也是无法测试网络断线的原因。

对比：有无 Mock.js 的请求流程

场景 无 Mock.js 有 Mock.js
网络正常 发送真实请求 → 返回后端数据 拦截请求 → 返回模拟数据
网络断开 发送请求 → 报错（可测试断线） 拦截请求 → 返回模拟数据（无法测试断线）

四、如何测试网络断线

4.1 方案一：条件性启用 Mock（推荐）

通过 URL 参数控制是否启用 mock，需要测试网络断线时，在 URL 中添加参数禁用 mock。

// mock.js

import Mock from 'mockjs';

const baseURL = 'http://your-api.com';

// 通过 URL 参数控制是否启用 mock
const isTestNetwork = new URLSearchParams(window.location.search).has('testNetwork');

// 测试网络时禁用 mock
if (!isTestNetwork) {
    Mock.mock(baseURL + "/api/getData.ashx?params=1", "post", (options) => {
        return Mock.mock({
            "code": "200",
            "return": "ok",
            "data|3": [{
                "id|+1": 1001,
                "name|+1": ["选项一", "选项二", "选项三"]
            }]
        });
    });
}

使用方式

访问地址 效果
http://localhost:5000 Mock 生效，返回模拟数据
http://localhost:5000?testNetwork Mock 禁用，发送真实请求，可测试断线
测试步骤：
1. 访问 http://localhost:5000?testNetwork
2. 打开 Chrome DevTools (F12) → Network → 选择 Offline （默认是throttling，在Disable Cache右边)
3. 发起请求，即可测试网络断线场景

4.2 方案二：用 Mock 模拟错误响应

不需要真正断网，使用 Mock.js 模拟各种错误状态。

import Mock from 'mockjs';

// 模拟正常响应
Mock.mock('/api/user', 'post', (options) => {
    return {
        code: 200,
        return: 'ok',
        data: { name: '测试用户' }
    };
});

// 模拟超时
Mock.mock('/api/user?params=testTimeout', 'post', (options) => {
    const error = new Error('timeout of 5000ms exceeded');
    error.code = 'ECONNABORTED';
    throw error;  // 抛出错误，axios 会捕获
});

// 模拟网络断开
Mock.mock('/api/user?params=testOffline', 'post', (options) => {
    const error = new Error('Network Error');
    error.code = 'ERR_NETWORK';
    throw error;
});

// 模拟服务器错误（500）
Mock.mock('/api/user?params=testServerError', 'post', (options) => {
    return {
        status: 500,
        message: '服务器内部错误'
    };
});

// 模拟业务失败
Mock.mock('/api/user?params=testFail', 'post', (options) => {
    return {
        code: 200,
        return: 'fail',
        data: '操作失败，请重试'
    };
});

调用测试

// 测试超时
$axios.post('/api/user?params=testTimeout', {})
    .catch(err => console.log('超时测试:', err.message));

// 测试断网
$axios.post('/api/user?params=testOffline', {})
    .catch(err => console.log('断网测试:', err.message));

4.3 方案三：临时禁用 Mock

最简单的方式，直接注释掉 mock 相关代码。

// import './mock';  // 注释掉 mock 引入

// 然后用 Chrome DevTools → Network → Offline 测试

4.4 方案四：请求不存在的域名

请求一个不存在的域名，会立即触发网络错误。

// 请求不存在的域名，无需断网即可测试网络错误
$axios.post('https://this-domain-does-not-exist-12345.com/api/test', {})
    .then(res => console.log('成功:', res))
    .catch(err => console.log('网络错误:', err.message));

方案对比

方案 优点 缺点
方案一
条件性启用		 灵活切换，不影响正常开发 需要修改 mock 代码
方案二
模拟错误		 无需真正断网 只能模拟已知错误类型
方案三
临时禁用		 最简单直接 需要频繁注释/取消注释
方案四
不存在的域名		 快速测试，无需额外配置 只能测试网络错误，无法测试超时等场景

mock/index.js

import Mock from 'mockjs';

const baseURL = 'http://your-api.com';

// 通过 URL 参数控制是否启用 mock
const isTestNetwork = new URLSearchParams(window.location.search).has('testNetwork');

if (!isTestNetwork) {
    console.log('Mock 已启用');
    
    // 获取辅导方式列表
    Mock.mock(baseURL + "/jk/jk_sj/jk_fudao_fangshi.ashx?params=1", "post", (options) => {
        let body = JSON.parse(options.body);
        console.log('请求参数：', body);
        
        return Mock.mock({
            "code": "200",
            "return": "ok",
            "data|3": [{
                "fudao_fangshi_bh|+1": 1001,
                "fudao_fangshi_mc|+1": ["上门辅导", "在线辅导", "住家家教"]
            }]
        });
    });

    // 用户登录
    Mock.mock(baseURL + "/api/login.ashx", "post", (options) => {
        let body = JSON.parse(options.body);
        
        if (body.username === 'admin' && body.password === '123456') {
            return {
                code: 200,
                return: 'ok',
                data: {
                    token: 'mock-token-12345',
                    userInfo: {
                        id: 1,
                        name: '管理员',
                        role: 'admin'
                    }
                }
            };
        } else {
            return {
                code: 200,
                return: 'fail',
                data: '用户名或密码错误'
            };
        }
    });

    // 获取用户信息
    Mock.mock(baseURL + "/api/userinfo.ashx", "get", {
        code: 200,
        return: 'ok',
        data: {
            'id|1-100': 1,
            'name': '@cname',
            'age|18-60': 1,
            'email': '@email',
            'phone': /^1[3-9]\d{9}$/
        }
    });

} else {
    console.log('Mock 已禁用，发送真实请求');
}

api/public.js

import request from '@/utils/neatui-axios.js';

const baseURL = 'http://your-api.com';
const debug = true;
const method = 'post';

/**
 * 获取辅导方式列表
 */
export function getFudaoFangshi(vm, data) {
    return request({
        debug: debug,
        vue: vm,
        original: false,
        method: method,
        baseURL: baseURL,
        url: "/jk/jk_sj/jk_fudao_fangshi.ashx?params=1",
        data: data
    });
}

/**
 * 用户登录
 */
export function login(vm, data) {
    return request({
        debug: debug,
        vue: vm,
        original: false,
        method: method,
        baseURL: baseURL,
        url: "/api/login.ashx",
        data: data
    });
}

页面调用示例

import { getFudaoFangshi, login } from '@/api/public.js';

export default {
    methods: {
        // 获取数据
        async fetchData() {
            try {
                const res = await getFudaoFangshi(this, { action: 'list' });
                console.log('数据:', res);
            } catch (err) {
                console.log('错误:', err.message);
            }
        },
        
        // 登录
        async handleLogin() {
            try {
                const res = await login(this, {
                    username: 'admin',
                    password: '123456'
                });
                console.log('登录成功:', res);
            } catch (err) {
                console.log('登录失败:', err.message);
            }
        }
    }
};
